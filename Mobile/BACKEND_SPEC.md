# Backend Specification: Supabase Pivot

This blueprint has been updated to use **Supabase** (PostgreSQL-as-a-Service) for rapid development.

## 1. Database Schema (Supabase/PostgreSQL)

You can copy and paste this into the **Supabase SQL Editor**.

```sql
-- Enable UUID extension
create extension if not exists "uuid-ossp";

-- 1. Organizational Structure
create table party_committees (
    id bigint generated by default as identity primary key,
    ma_dang_bo text not null unique,
    ten_dang_bo text not null,
    cap_uy text,
    bi_thu_id bigint,
    pho_bi_thu_id bigint,
    so_chi_bo_truc_thuoc int default 0,
    tong_so_dang_vien int default 0,
    ngay_thanh_lap date,
    is_active boolean default true,
    is_deleted boolean default false,
    created_at timestamptz default now()
);

create table party_cells (
    id bigint generated by default as identity primary key,
    ma_chi_bo text not null unique,
    ten_chi_bo text not null,
    dang_bo_id bigint not null references party_committees(id),
    don_vi_truc_thuoc text,
    bi_thu_id bigint,
    so_dang_vien int default 0,
    is_active boolean default true,
    created_at timestamptz default now()
);

create table party_members (
    id bigint generated by default as identity primary key,
    ma_dang_vien text not null unique,
    ho_ten text not null,
    ngay_sinh date,
    gioi_tinh text,
    cmnd_cccd text unique,
    email text,
    so_dien_thoai text,
    dia_chi text,
    anh_dai_dien text,
    ngay_vao_dang date,
    ngay_chinh_thuc date,
    chi_bo_id bigint references party_cells(id),
    chuc_vu_dang text,
    trinh_do_hoc_van text,
    trinh_do_ly_luan_chinh_tri text,
    nghe_nghiep text,
    is_active boolean default true,
    is_deleted boolean default false,
    created_at timestamptz default now()
);

-- 2. Auth (Integrates with Supabase Auth)
-- Note: Supabase handles users in 'auth.users'. 
-- This table links our public profile to Supabase Auth.
create table profiles (
  id uuid references auth.users on delete cascade primary key,
  dang_vien_id bigint unique references party_members(id),
  vai_tro text not null default 'user',
  updated_at timestamptz default now()
);

-- 3. Meetings & Attendance
create table cell_meetings (
    id bigint generated by default as identity primary key,
    chi_bo_id bigint not null references party_cells(id),
    tieu_de text not null,
    noi_dung text,
    thoi_gian timestamptz not null,
    dia_diem text,
    chu_tri_id bigint references party_members(id),
    thu_ky_id bigint references party_members(id),
    trang_thai text default 'sap_dien_ra',
    created_at timestamptz default now()
);

create table meeting_attendance (
    id bigint generated by default as identity primary key,
    sinh_hoat_id bigint not null references cell_meetings(id),
    dang_vien_id bigint not null references party_members(id),
    trang_thai text default 'vang',
    checkin_at timestamptz,
    created_at timestamptz default now()
);

-- 4. Finance (Union Fees)
create table union_fees (
    id bigint generated by default as identity primary key,
    dang_vien_id bigint not null references party_members(id),
    thang_nam date not null,
    so_tien numeric(18, 0) not null,
    trang_thai text default 'chua_dong',
    min_chung_anh text,
    created_at timestamptz default now()
);

-- 5. News & Documents
create table news (
    id uuid default uuid_generate_v4() primary key,
    title text not null,
    summary text,
    content text,
    thumbnail_url text,
    category_id bigint,
    status int default 1,
    published_at timestamptz default now()
);

create table documents (
    id bigint generated by default as identity primary key,
    tieu_de text not null,
    duong_dan_file text not null,
    category_id bigint,
    created_at timestamptz default now()
);
```

## 2. Supabase Integration Strategy

### Option A: Supabase Client (Recommended for Speed)
Instead of a custom .NET/Node API, use `@supabase/supabase-js`. 
- **Pros**: Zero backend code. Real-time updates. Automatic REST API.
- **Cons**: Requires refactoring frontend services (replacing Axios).

### Option B: API Proxy (Edge Functions)
Keep existing `/api/...` structure and use Supabase Edge Functions to handle the logic.
- **Pros**: Minimal frontend changes.
- **Cons**: Requires writing Deno/TypeScript functions for each endpoint.

## 3. Supabase Specifics

### 3.1 Storage Buckets
Create the following buckets in the Supabase Storage tab:
- `avatars`: Publicly readable, restricted write (user only).
- `documents`: Publicly readable (or restricted to authenticated), admin write.
- `union-fees`: Secure bucket for payment proof images.

### 3.2 Row Level Security (RLS) Example
```sql
-- Allow members to see only their own profile
alter table party_members enable row level security;
create policy "Users can view their own profile"
  on party_members for select
  using ( auth.uid() = cmnd_cccd ); -- Or link via profiles table
```
